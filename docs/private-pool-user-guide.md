# 私人号池使用说明

私人号池为每个 new-api 用户提供一套独立的 CLIProxyAPI 实例、账号目录、路由分组和内部渠道。普通用户只能访问自己的池；超级管理员可以在「号池管理」中查看和管理全部池。

## 用户操作流程

1. 登录后进入「我的号池」，点击「创建我的私人号池」。每个用户只能创建一个，系统最多允许 10 个私人池。
2. 等待页面显示「就绪」。开池过程由宿主机 watcher 创建独立容器，通常无需手动刷新。
3. 粘贴 Codex auth JSON，或导入 ZIP。私人池最多 20 个账号；ZIP 最大 8 MiB、最多读取 50 个 JSON，每个 JSON 最大 512 KiB。
4. 对至少一个账号执行验活，确认其状态正常。平台的定时清理、定时额度刷新和自动重置不会操作私人池；用户可在页面手动刷新额度或消耗重置券。
5. 进入「API Keys」创建密钥。普通用户创建的密钥会由后端强制绑定到自己的私人池，并关闭跨分组重试；即使前端请求被篡改，也不能选公共池或其他用户的池。

## 隔离与失败行为

- 私人池使用不可变路由分组 `private-<user_id>`，不会加入全局可用分组列表。
- 管理接口不接受客户端传入的池 ID，而是根据当前登录用户在服务端解析归属。
- 用户列表响应会移除文件路径、auth index、access/refresh token、原始 JWT 和其他凭证字段。
- 私人池不存在、未就绪或不可用时，API Key 创建/调用直接失败，不会回退到公共号池。
- 普通用户不能删除整个池、改名、批量自动清理或操作其他用户的账号；超级管理员保留全局管理能力。

## 超级管理员视角

「号池管理」只对超级管理员开放。私人池标签下会显示所有者用户名（用户名不可用时显示用户 ID），并可执行现有的账号管理、改名和删除操作。删除池时账号目录按现有 watcher 策略保留在宿主机，便于人工恢复。

## 部署资源边界

私人池使用 `deploy/config.private.example.yaml` 轻量模板。watcher 为每个实例设置：

- 内存上限 256 MiB，reservation 64 MiB；
- CPU 上限 0.75，PID 上限 128；
- 应用日志总量 32 MiB、错误日志最多 3 个；
- Docker 日志每文件 10 MiB、最多 2 个。

这些参数适合当前约 10 个用户、日活 3–4 人的小规模部署；后续若单池并发明显提高，可只调整私人模板和 watcher 资源参数，不需要改变权限与路由模型。
