**T3.1 — PoolEntry / PoolInfo 带 BuildMode 字段**(`server/newapi/common/xju_pool_registry_test.go`,testify)

**T3.2 — ListConfiguredPools 回填 build_mode 默认值**(同文件)
- default/k12 池:`BuildMode=="cliproxy"`。
- 动态池 `BuildMode` 为空时:`ListConfiguredPools` 返回的 `PoolInfo.BuildMode=="cliproxy"`(向后兼容:老 registry 无该字段的条目视作 cliproxy)。

**T3.3 — normalizeBuildMode 归一**(`server/newapi/service/xju_pool_provision_test.go`,testify 表测)
- `"gopool"`/`" GoPool "`/`"GOPOOL"` → `"gopool"`;`""`/`"cliproxy"`/`"garbage"`/`"xyz"` → `"cliproxy"`(未知一律降级默认,不报错)。

**T3.4 — RequestPoolProvision 把 mode 写进请求 + 记入待定表**(同文件,扩展既有 `TestPoolProvisionFlow`)

**T3.6 — controller 读取 mode**(逻辑随 T3.4/T3.5 覆盖;controller 仅透传)

## ⑤ 前端订阅日期 epoch 兜底(P1)—— bun test

> 前置重构:把 `subscriptionUntil` / `isSubscriptionExpired` 从 `index.tsx` 抽到 `web/src/features/pool/subscription.ts`(仿 dashboard/lib/flow.ts 可测模式),`index.tsx` 改为 import。测试:`web/src/features/pool/subscription.test.ts`,`bun test src/features/pool/subscription.test.ts`。

**T5.1 — ISO-8601 字符串正常解析(现状不回归)**
- `subscriptionUntil({id_token:{chatgpt_subscription_active_until:'2027-06-01T00:00:00Z'}})` → 非 null,`getUTCFullYear()===2027`。

**T5.2 — 数值 Unix 秒(10 位)按秒解析,不落 1970**
- `chatgpt_subscription_active_until: 1811808000`(=2027-06-01,秒)→ `getUTCFullYear()===2027`(而非 1970)。

**T5.3 — 数值毫秒(13 位)按毫秒解析**
- `chatgpt_subscription_active_until: 1811808000000` → `getUTCFullYear()===2027`。

**T5.4 — 缺失/空/非法 → null**
- `undefined` / `''` / `'not-a-date'` / `{}`(无 id_token)→ 全部返回 `null`。

**T5.5 — isSubscriptionExpired 语义**
- 过去日期(`'2000-01-01T00:00:00Z'`)→ `true`;远未来(`'2099-01-01T00:00:00Z'`)→ `false`;`null` 来源(无日期)→ `false`(无日期不等于过期)。

**验收:** `bun test src/features/pool/subscription.test.ts` 全绿;`bun run typecheck` 清零;`index.tsx` 行为不变(徽章渲染视觉无回归)。
